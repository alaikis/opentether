## 1. 项目概述

### 1.1 项目背景

本项目旨在开发一个企业级智能体（AI Agent）系统，基于 Go + Fiber + GORM 技术栈构建。该系统需要支持：

- 多用户隔离的记忆系统
- 嵌入式 Web 管理端
- SSO 员工导入 + API 用户推送
- 多平台 IM 集成（企业微信、飞书、钉钉、WhatsApp）
- Text2SQL 及智能数据分析
- Skills 配置与管理
- 企业级权限控制与审计

### 1.2 设计原则

1. **模块化单体架构** - 当前采用单体架构，支持未来平滑拆分
2. **本地化部署** - 减少外部依赖，支持完全离线运行
3. **权限边界控制** - 系统只做规划范围内的操作
4. **多语言友好** - 支持企业出海需求
5. **可扩展性** - 支持 Skills、Tools、执行器的扩展

### 1.3 约束条件

- **架构**: 模块化单体应用，支持独立/内嵌执行器
- **部署模式**: 单一二进制、Docker、Kubernetes
- **用户规模**: 100 - 10,000+ 用户
- **数据库**: SQLite（内置）、MySQL、PostgreSQL 可选
- **向量数据库**: 内存向量库（内置）、Qdrant（可选）

## 3. 核心模块设计

### 3.1 统一网关 (Unified Gateway)

统一网关是所有请求的入口，负责：

- **请求路由**: 根据来源识别请求类型 (Web/IM/API/Task)
- **协议转换**: REST / WebSocket / Callback
- **认证鉴权**: JWT Token / API Key / IM 回调验证
- **限流熔断**: 防止系统过载
- **上下文管理**: 加载用户/组上下文
- **流程编排**: 协调意图识别、规划、执行

### 3.2 智能体核心 (AI Agent Core)

#### 3.2.1 意图引擎 (Intent Engine)

- 向量匹配 + LLM 确认的混合方式
- 支持多语言意图识别
- Few-shot 学习支持

#### 3.2.2 规划引擎 (Planning Engine)

- 目标分解与任务规划
- 路径优化与约束检查
- **权限边界检查**:
  - Skills 白名单校验
  - 数据访问权限校验
  - 操作边界校验
  - 审批流程触发

#### 3.2.3 执行引擎 (Execution Engine)

- 任务执行器
- 工具调用器
- 结果聚合与错误处理

### 3.3 记忆系统 (Memory System)

| 记忆类型 | 作用域 | 说明 |
|---------|--------|------|
| Session Memory | 单次会话 | 每次对话独立，不保存 |
| User Memory | 用户级 | 用户有独立的长期记忆 |
| Group Memory | 用户组 | 组内共享记忆，组间隔离 |

## 6. Skills 配置体系

### 6.1 Skills 类型

| 类型 | 说明 | 示例 |

### 6.2 Skills 与用户组绑定

Skills 需要与用户组绑定，用户只能使用其所属用户组授权的 Skills。

```yaml
# Skill 配置示例
skill:
  id: "skill_text2sql"
  name: "Text2SQL"
  access_control:
    allowed_groups: ["group_analyst", "group_admin"]
    denied_groups: []
    require_approval: false
    data_scope: "all"
```

### 6.3 权限分配权限

支持配置特定用户或用户组拥有分配 Skills 的权限：

- **角色配置**: 创建 skill_manager 角色
- **用户组配置**: 指定用户组为技能管理员
- **数据范围**: 限制可分配的 Skills 范围

- 创建/编辑/删除 Skills
- 配置 Skills 参数
- 分配 Skills 给用户组
- 测试 Skills
- 同步向量索引

```yaml
      type: "redis"
      address: "localhost:6379"
```

## 10. 安全设计

### 10.1 安全措施

| 措施 | 说明 |
|------|------|
| 密码存储 | bcrypt/argon2 哈希 |
| 敏感数据加密 | AES-256-GCM |

| 认证授权 | JWT (RS256) + API Key |

| 速率限制 | 全局/用户/IP 限流 |
| 敏感数据脱敏 | 手机号/身份证/银行卡 |

```yaml
  encryption:
    key: "${ENCRYPTION_KEY}"

    enabled: true
```
