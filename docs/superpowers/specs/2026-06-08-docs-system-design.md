# 文档管理系统设计规格

## 1. 项目概述

- **项目名称**: OpenTether 文档系统
- **项目类型**: 系统内置文档管理模块
- **核心功能**: 为管理员和用户提供开发文档和用户手册的查看、搜索和导航功能
- **目标用户**: 系统管理员、开发人员、普通用户

## 2. 需求概述

### 2.1 功能需求

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| F01 | 文档分类展示（开发文档/用户手册两大类） | P0 |
| F02 | 功能模块细分（IM配置、LLM集成、技能系统、任务调度） | P0 |
| F03 | Markdown 渲染（支持代码高亮） | P0 |
| F04 | 侧边栏目录导航（可折叠） | P0 |
| F05 | 文档内搜索功能 | P0 |
| F06 | 文档目录自动生成（从 H1/H2 标题提取） | P1 |

### 2.2 非功能需求

- 文档为系统内置，只读不可编辑
- 页面加载时间 < 2秒
- 搜索响应时间 < 500ms
- 兼容现代浏览器（Chrome、Firefox、Safari、Edge）

## 3. 技术方案

### 3.1 架构设计

采用**混合方案**：
- 静态 Markdown 文件存储文档内容
- 后端提供搜索索引 API
- 前端负责渲染和交互

### 3.2 目录结构

```
admin-ui/build/
├── docs/
│   ├── index.html          # 文档首页
│   ├── search.json         # 搜索索引
│   ├── dev/                # 开发文档
│   │   ├── im.md
│   │   ├── llm.md
│   │   ├── skills.md
│   │   └── scheduler.md
│   └── user/               # 用户手册
│       ├── im.md
│       ├── llm.md
│       ├── skills.md
│       ├── scheduler.md
│       └── faq.md
```

### 3.3 后端 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/docs/search` | GET | 搜索文档（?q=关键词） |
| `/api/v1/docs/categories` | GET | 获取分类结构 |

### 3.4 前端技术选型

| 功能 | 技术方案 |
|------|----------|
| Markdown 渲染 | marked.js |
| 代码高亮 | highlight.js |
| 搜索 | Fuse.js |
| 目录生成 | 自定义脚本 |

## 4. 页面设计

### 4.1 布局结构

```
┌─────────────────────────────────────────────────────────────────┐
│  Header: OpenTether 文档 + 搜索框 + 分类切换（开发/用户）        │
├────────────────┬────────────────────────────────────────────────┤
│                │                                                 │
│  Sidebar:      │  Content:                                       │
│  - 开发文档     │  - 文档标题 (H1)                                │
│    - IM配置    │  - 文档内容 (Markdown 渲染)                      │
│    - LLM集成   │  - 代码块 (语法高亮)                             │
│    - 技能系统   │  - 子标题 (H2) → 目录跳转                       │
│    - 任务调度  │                                                 │
│  - 用户手册     │                                                 │
│    - IM配置    │                                                 │
│    - LLM集成   │                                                 │
│    - 技能系统   │                                                 │
│    - 任务调度  │                                                 │
│    - 常见问题  │                                                 │
│                │                                                 │
├────────────────┴────────────────────────────────────────────────┤
│  TOC: 目录导航（自动从当前文档 H1/H2 生成，可点击跳转）           │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 交互设计

- 侧边栏分类可折叠/展开
- 点击目录项跳转到对应标题
- 搜索支持模糊匹配
- 分类切换无刷新更新内容

## 6. 实现计划

### Phase 1: 后端 API 开发
- 创建文档 API 路由
- 实现搜索接口
- 实现分类接口

### Phase 2: 前端页面开发
- 创建 docs/index.html 页面
- 集成 marked.js 和 highlight.js
- 实现侧边栏导航组件
- 实现目录导航组件
- 集成 Fuse.js 搜索

### Phase 3: 文档内容创建
- 编写开发文档（IM、LLM、Skills、Scheduler）
- 编写用户手册（IM、LLM、Skills、Scheduler、FAQ）

### Phase 4: 测试和优化
- 功能测试
- 性能优化
- UI 调整

## 7. 验收标准

- [ ] 文档页面可正常访问
- [ ] 分类切换正常（开发文档/用户手册）
- [ ] 侧边栏目录可折叠/展开
- [ ] Markdown 正确渲染，代码高亮正常
- [ ] 目录导航可点击跳转
- [ ] 搜索功能正常工作
- [ ] 所有文档内容完整

## 8. 备注

- 后续可扩展：版本管理、更多模块文档
